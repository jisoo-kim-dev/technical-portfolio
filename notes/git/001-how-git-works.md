# How Git Works

## What is Git?

Git is a distributed version control system.

It tracks changes in files, allowing developers to collaborate, restore previous versions, and manage software projects efficiently.

---

## Local vs Remote Repository

### Local Repository

Stored on my computer.

Changes are not visible to others until they are pushed.

### Remote Repository

Stored on GitHub.

Used for backup, collaboration, and sharing projects.

---

## Basic Git Workflow

1. Modify files
2. Save changes
3. Commit changes
4. Push to GitHub

```
Working Directory
        ↓
      Commit
        ↓
 Local Repository
        ↓
     Push
        ↓
 GitHub Repository
```

---

## Common Git Terms

| Term | Meaning |
|------|---------|
| Repository | Project folder tracked by Git |
| Commit | Snapshot of changes |
| Push | Upload commits to GitHub |
| Pull | Download newest changes |
| Clone | Copy repository to local computer |
| Branch | Separate line of development |
| Merge | Combine branches |

---

## What I Learned

- Git records project history.
- Commit saves changes locally.
- Push uploads commits to GitHub.
- GitHub stores the remote repository.
